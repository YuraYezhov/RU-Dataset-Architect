# AI usage log

## Использованные инструменты
* **Gemini 3 Flash Preview (чат-бот):** Основной ассистент на протяжении всего проекта. Использовался для мозгового штурма, проектирования архитектуры системы, получения советов по методам OpenCV и логике обработки данных.
* **Gemini 3.1 Flash Lite (интегрирована в Continue):** Использовалась непосредственно в среде разработки для генерации шаблонного кода, отладки функций и помощи в написании документации.
* **GLM-5.2 (ИИ-агент):** Специализированное использование для автоматизации рутинной обработки текстовых данных при формировании файла labels.csv на 100 записей.
* **VS Code + Continue:** Основная среда разработки, позволившая объединить процесс написания кода на Python и взаимодействие с ИИ-агентом через контекстное меню и чат.

## Для чего использовались
* **Выбор темы датасета:** Совместный мозговой штурм по выбору предметной области. В ходе дискуссии были отклонены идеи с монетами и сложной архитектурой из-за трудностей контроля условий съемки. Выбрана тема рукописных прописей как идеального «стерильного» материала (без шумов и искажений).
* **Проектирование признаков:** ИИ выступил консультантом при выборе ключевых дескрипторов (aspect_ratio, density, peaks), которые впоследствии были доработаны и реализованы вручную с использованием библиотеки OpenCV.
* **Написание скриптов:** Генерация базового каркаса класса `HandwritingExtractor` и логики пакетной обработки данных с использованием модулей `pathlib` для файловой системы и `pandas` для формирования итогового датасета.
* **Написание примеров на PascalABC.NET:** Генерация трех демонстрационных программ (`01_load_dataset.pas`, `02_describe_dataset.pas`, `03_baseline_model.pas`) для демонстрации работы с датасетом через библиотеку MLABC.
* **Подготовка документации:** Помощь в структурировании и наполнении файлов `handwriting_minimal_dataset.meta`, `quality_report.md`, `sources.md` и `ai_log.md`.

## Что проверено вручную
* **Логика OpenCV:** Поскольку алгоритмы извлечения признаков писались вручную, проведена их полная верификация через тестовый скрипт `test_features.py`.
* **Логика разметки:** Проведена сверка соответствия количества букв в словах (согласно `labels.csv`) с данными в колонке `target_letters`.
* **Архитектура:** Самостоятельное принятие решения о разделении кода на два независимых модуля (логика и менеджер) для повышения читаемости проекта.
* **Разумность значений:** Сверка итогового CSV-файла на предмет отсутствия пустых значений (NaN) и и проверка разумности вычисленных коэффициентов.
* **Качество текстов:** Ручная проверка и коррекция всех текстов документации, генерируемых ИИ, на соответствие фактическим действиям и техническим терминам.
