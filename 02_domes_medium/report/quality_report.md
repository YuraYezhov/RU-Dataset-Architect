## Рекомендованная обработка
Для данного датасета на этапе генерации Easy-версии выполнена первичная очистка.
* **Очистка:** Для Easy-версии проведена фильтрация по порогу `aspect_ratio` (0.2-4.0) и удаление строк с отсутствующими данными (NaN).
* **Заполнение пропусков:** не требуется (рекомендуется удаление строк).
* **Балансировка:** Применение техник Oversampling или `class_weight='balanced'` рекомендуется только при значительном увеличении объема данных для редких классов, чтобы избежать переобучения.
* **Нормализация (Scaling):** Для алгоритмов, основанных на расчете геометрического расстояния между точками (**k-NN**), а также для линейных моделей и нейронных сетей, **обязательно** выполнение масштабирования признаков (StandardScaler или Min-Max Scaling). В предоставленном примере обучения k-NN масштабирование реализовано через `Pipeline`. Для алгоритмов на основе деревьев решений (Random Forest) нормализация не требуется.
