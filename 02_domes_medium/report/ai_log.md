# AI usage log

## Использованные инструменты
*   **Gemini 3 Flash Preview (чат-бот):** Основной ассистент для высокоуровневого планирования. Использовался для проектирования архитектуры парсинга сайта sobory.ru, обсуждения логики «умного кроппинга» куполов и выбора геометрических признаков.
*   **Gemini 3.1 Flash Lite (интегрирована в Continue):** Использовалась в VS Code для написания регулярных выражений при парсинге, отладки OpenCV-фильтров и генерации структуры итоговых CSV-файлов.
*   **VS Code + Continue:** Среда разработки, обеспечившая бесшовную интеграцию Python-скриптов обработки изображений и ИИ-подсказок.

## Для чего использовались
* **Выбор темы датасета:** Совместный мозговой штурм по выбору предметной области. Выбрана тема куполов как объектов с четкой геометрической типологией (луковица, шлем, шатер).
* **Проектирование признаков:** Консультации по выбору метрик, инвариантных к масштабу и освещению. Выбраны коэффициенты aspect_ratio, solidity, extent, top_width_ratio и моменты Ху.
* **Написание скриптов:** Генерация кода для многоэтапного парсинга, скрипта пакетной загрузки изображений и основного класса DomesExtractor.
* **Написание примеров на PascalABC.NET:** Генерация трех демонстрационных программ (`01_load_dataset.pas`, `02_describe_dataset.pas`, `03_baseline_model.pas`) для демонстрации работы с датасетом через библиотеку MLABC.
* **Подготовка документации:** Помощь в структурировании и наполнении файлов `domes_medium_dataset.meta`, `quality_report.md`, `sources.md` и `ai_log.md`.

## Что проверено вручную
*   **Качество парсинга:** Ручная проверка 8 промежуточных JSON-файлов на предмет корректности ссылок и ID объектов.
* **Визуальный контроль:** Проведен выборочный ручный контроль отрисовки контуров поверх куполов на тестовых изображениях для подтверждения корректности работы алгоритма сегментации и аппроксимации формы.
* **Архитектура и логика:** Проверена архитектура системы и логика скриптов в файле `create_dataset` на предмет корректности сопоставления идентификаторов объектов из реестра с именами файлов изображений, а также устойчивости кода к пропускам в данных.
* **Качество текстов:** Ручная проверка и коррекция всех текстов документации, генерируемых ИИ, на соответствие фактическим действиям и техническим терминам.
