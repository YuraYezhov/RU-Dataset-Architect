# Data sources

Для формирования датасета **DomesGeometry (Medium)** были использованы следующие открытые данные:

**Источник:** Народный каталог православной архитектуры [sobory.ru](https://sobory.ru/).
**Дата обращения:** 08.07.2026.
**Лицензия:** Использование материалов в некоммерческих образовательных целях (согласно правилам цитирования и использования материалов сайта sobory.ru).

## Выполненные преобразования

* **Автоматизированный сбор данных:** Реализован многоэтапный процесс парсинга: были собраны метаданные и ссылки на изображения для объектов из 8 городов Золотого кольца (Сергиев Посад, Переславль-Залесский, Ростов Великий, Кострома, Иваново, Суздаль, Ярославль, Владимир). Информация была структурирована в 8 региональных JSON-файлах.
* **Подготовка реестра и загрузка:** Выполнено слияние региональных данных в единый структурированный реестр all_domes.json. На его основе реализован скрипт пакетной загрузки: для каждого объекта (id) скачивалось до 3-х ракурсов. Пустые ссылки и отсутствующие изображения в реестре помечались как null для сохранения целостности структуры. Файлы сохранялись в формате {id}_{index}.jpg.
* **Извлечение признаков (Computer Vision):** С помощью библиотеки OpenCV реализован класс `DomesExtractor`. Процесс обработки каждого изображения включает:
    *   Преобразование в оттенки серого и Гауссово размытие (ядро 5x5) для фильтрации шумов;
    *   Бинаризацию по методу Оцу;
    *   Поиск внешних контуров и выбор наиболее крупного (доминирующего) фрагмента;
    * Аппроксимацию формы: использование функции выпуклой оболочки для получения сглаженного силуэта купола, устойчивого к бликам и теням.
    *  Рассчитаны 5 геометрических метрик: 
        * отношение ширины к высоте (`aspect_ratio`)
        * плотность/выпуклость (`solidity`)
        * заполненность (`extent`)
        * коэффициент остроты вершины (`top_width_ratio`)
        * первый момент Ху (`hu_moment_1`).
* **Подготовка разметки:**
    * Век постройки (`century`): Автоматически извлечен из текстового поля «Год постройки» с последующей конвертацией в числовой формат века.
    *   **Категория формы (`target_shape`):** Проведена полная ручная разметка всех объектов в исходных JSON-файлах. На основе визуального анализа фотографий каждому объекту был присвоен один из трех целевых классов (`onion`, `helmet`, `tent`). В случаях, когда на снимках отсутствовал купол или объект относился к иным архитектурным типам, поле было заполнено значением `null`. Это позволило на этапе сборки Easy-версии автоматически исключить объекты, не соответствующие целям исследования.
* **Финальная сборка:** Итоговая таблица сформирована путем сопоставления данных из all_domes.json с результатами работы OpenCV-экстрактора. Подготовлено две версии данных:
    * **Raw:** Базовый датасет (861 строка), содержащий все результаты парсинга, включая пропуски (NaN), ошибки распознавания контуров и выбросы.
    * **Easy:** Очищенная версия (560 строк), из которой удалены записи с отсутствующими изображениями, пропущенными метками классов и аномальными значениями метрик.
