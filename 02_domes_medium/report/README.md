# DomesGeometry (Medium)

## Цель
Автоматическая классификация типов куполов архитектурных объектов по их геометрическим характеристикам. Датасет предназначен для отработки алгоритмов многоклассовой классификации и методов интеллектуального кроппинга объектов в задачах компьютерного зрения. !

### Предметная область
Изображения православной архитектуры Золотого кольца (церкви, часовни, колокольни), содержащие три основных типа куполов: луковичные (onion), шлемовидные (helmet) и шатровые (tent).

## Файлы
* `data/domes_medium_dataset_raw.csv` — базовая версия датасета со всеми результатами парсинга и шумами (861 объект).
* `data/domes_medium_dataset_easy.csv` — очищенная версия датасета с удаленными выбросами и подтвержденной разметкой (560 объектов).
* `data/intermediate_data/all_domes.json` — объединенный реестр метаданных и ссылок на изображения.
* `meta/domes_medium_dataset.meta` — технический паспорт датасета.
* `samples/01_load_dataset.pas` — пример загрузки и вывода данных в PascalABC.NET.
* `samples/02_describe_dataset.pas` — пример вывода схемы данных и первичного анализа.
* `samples/03_baseline_model.pas` — пример построения простейшей модели линейной регрессии.

## Задачи машинного обучения
Классификация: определение формы купола (target_shape).

### Признаки
| Имя | Тип | Описание | Роль |
| :--- | :--- | :--- | :--- |
| `file_name` | string | Идентификатор файла изображения | Технический |
| `aspect_ratio` | float | Отношение ширины к высоте контура | Признак |
| `solidity` | float | Степень выпуклости формы | Признак |
| `extent` | float | Заполненность описывающего прямоугольника | Признак |
| `top_width_ratio` | float | Отношение ширины вершины к максимальной ширине | Признак |
| `hu_moment_1` | float | Первый инвариантный момент Ху | Признак |
| `century` | int | Век постройки объекта | Признак |
| `target_shape` | string | Тип формы (onion, helmet, tent) | Целевая переменная |

## Целевая переменная
`target_shape` — категориальная переменная, принимающая одно из трех значений, соответствующих архитектурному типу купола.

## Особенности данных
В Raw-версии присутствуют пропуски и выбросы, вызванные сложностью сегментации объектов на сложном фоне. Имеется значительный дисбаланс классов в пользу луковичных куполов. Категориальный признак century используется для учета исторического контекста.

## Использование
Данные могут быть загружены в среду PascalABC.NET

## Ограничения
Алгоритм извлечения признаков чувствителен к перекрытиям объектов (деревья, леса) и ракурсу съемки. Easy-версия ограничена наиболее четкими кадрами, что упрощает задачу обучения, но снижает репрезентативность для "грязных" реальных данных.
