# AI usage log

## Использованные инструменты
* **Gemini 3 Flash Preview (чат-бот):** Основной ассистент на протяжении всего проекта. Использовался для мозгового штурма, проектирования архитектуры системы, анализа стратегий обработки пропущенных данных (NaN) и выбора методов импутации.
* **GLM-5.2 (ИИ-агент):** Специализированный инструмент для глубокого анализа сырых данных. Использовался для автоматического выявления единиц измерения в тексте, проектирования словаря основ слов (`keywords_dict_v2.py`) и отладки регулярных выражений для парсинга веса и габаритов.
* **Gemini 3.1 Flash Lite (интегрирована в Continue):** Использовалась непосредственно в среде разработки для генерации шаблонного кода, отладки функций и помощи в написании технической документации.
* **VS Code + Continue:** Основная среда разработки, позволившая объединить процесс написания кода на Python и взаимодействие с ИИ-агентом через контекстное меню и чат.

## Для чего использовались
* **Выбор темы датасета:** Совместный мозговой штурм по выбору предметной области. Выбрана тема реестра утраченных культурных ценностей как наиболее масштабная и демонстрирующая навыки работы с "грязными" данными.
* **Проектирование признаков:** ИИ выступил консультантом при выборе ключевых дескрипторов, которые впоследствии были реализованы через автоматизированный поиск подстрок и регулярные выражения.
* **Написание скриптов:** Генерация базового каркаса логики пакетной фильтрации 26 000 записей и использования модуля `pandas` для формирования версий датасета (Raw, Easy).
* **Написание примеров на PascalABC.NET:** Генерация четырех демонстрационных программ (`01_load_dataset.pas`, `02_describe_dataset.pas`, `03_DecisionTreeClassifier_model.pas`, `03_RandomForestClassifier_model.pas`) для демонстрации работы с датасетом через библиотеку MLABC.
* **Подготовка документации:** Помощь в структурировании и наполнении файлов `culture_large_dataset.meta`, `quality_report.md`, `sources.md` и `ai_log.md`.

## Что проверено вручную
* **Логика регулярных выражений:** Проведена верификация корректности извлечения чисел из строк типа "180 см", "7,5 гр" через тестовый скрипт.
* **Точность фильтрации:** Проверка правильности распределения объектов по 4 целевым группам (ART, ICON, NUMISMATICS, DOCUMENTS) после срабатывания текстовых хинтов.
* **Логика разметки:** Проведена сверка соответствия исходных категорий реестра итоговым числовым меткам в колонке `target_label`.
* **Разумность значений:** Сверка итогового CSV-файла на предмет отсутствия пустых значений (NaN) после глобальной импутации и проверка адекватности медианных размеров объектов.
* **Качество текстов:** Ручная проверка и коррекция всех текстов документации, генерируемых ИИ, на соответствие фактическим действиям и техническим терминам.
